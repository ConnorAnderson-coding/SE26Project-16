# 推荐分项打分实验

## 1. 实验目标

复现当前推荐管线（用户偏好向量 → ES kNN → 硬过滤 → 候选内 min-max → 加权求和），导出不同用户对各活动的 **原始分 / 归一化分 / 加权贡献 / 最终分**，定位「兴趣差异大的用户推荐列表却高度相似」等问题根因。

## 2. 实验环境与方案

- 索引：`campus_activities`，嵌入模型：`campus_gte`
- 偏好向量：最近 K=10 条报名 embedding，半衰期 30 天；兴趣文本推理后凸组合 `0.4·v_int + 0.6·v_hist`
- 召回：kNN k=50，仅 `status=published`
- 硬过滤：已报名活动 + 与已报名时段重叠
- 权重：sim=0.55, tag=0.15, social=0.15, hot=0.1, time=0.05 （对候选集做 min-max；time 已是 0/1）
- 社交：`s(A,B)=log(1+c_co)+0.8·log(1+c_A→B)+0.8·log(1+c_B→A)`
- 复现：`python backend/scripts/recommend-score-experiment.py`
- 生成时间：2026-07-14T18:38:25

## 5. 问题诊断（基于本次实验结果）

### 5.1 【主因】候选内 min-max 在窄分布 / 常数分布下失真

- **sim**：张三过滤后候选 span 仅 **0.0185**（σ=0.0064）。min-max 把噪声级差距放大成 0～1，**0.55 权重不再表示绝对相关度**，只表示池内相对名次。
- **tag（实现缺陷）**：张三剩余 8 个候选的 `tag_raw` **全部为 0**（兴趣标签无一命中），但 `VectorMath.minMaxNormalize` 在 `max<=min` 时返回 **1.0**，于是每人白得 `c_tag=0.15`。该项对排序无区分度，却虚高 final，掩盖「零标签匹配」。
- 篮球对张三：`sim_raw≈0.93`（池内最高 → sim_n=1）+ `soc_n=1`（李四主办）+ `tag_n=1`（伪满分）→ final≈0.93，冲到 Top1。

### 5.2 硬过滤掏空「最对口」活动

- 张三已报名 ids：`[1, 2, 3, 4, 5]` （AI 讲座、羽毛球、摄影、志愿、程序训练营 —— 正中兴趣）。
- 另有 4 条因与已报活动**时间重叠**被剔除（创业路演、机器人、马拉松等）。过滤后仅剩 ~8 个主题离散候选，相对排序更易被社交/热度左右。

### 5.3 社交分偏向稀疏主办方图

- 张三侧社交强度：`524030910002` s=3.0428；`T001` s=2.1266
- 对李四 s=3.0428（co=4），`soc_n=1` 单独贡献 0.1500，把「李四办的篮球/桌游」整体抬一档。
- seed 主办方几乎只有 T001 / 张三 / 李四，社交项近似「是否报过李四/老师活动」的阶跃特征。

### 5.4 热度与时间

- hot 权重 0.1：报名+收藏高的活动对所有用户同向偏置。
- time 权重 0.05：张三含 weekday_evening，篮球（工作日下午）`time_fit=0` 仍因其他项过高排第一 —— 说明辅因子无法挡住失真的主项。

### 5.5 为何张三≈李四：机制链

```text
兴趣不同 → 偏好向量不同 → kNN 原始序本应不同
        ↓ 硬过滤去掉各自已报的「强匹配」
剩余候选 sim 挤在 ~0.91–0.93 → min-max 放大噪声
```
